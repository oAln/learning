1). Symbols

Symbols are new primitive data type introduced in ES6 which represents a unique identifier.

Symbols are immutable (cannot be changed) and are unique.

// id is a symbol with the description "id"

// define => const symbol = Symbol('value')

// check description => symbol.description => 'value'


let id = Symbol("id");
let user = {{'{'}} name: "John" {{'}'}}
user[id] = 1; // not "id": 1
By default, only two primitive types serve as object keys:

1. String type

2. Symbol type.

If two scripts(code snippet) has the same object identifier(key) and then there will be no conflict between them as symbols are different, even if they have the same name but if we used a string instead of a symbol, then there would be a conflict.
